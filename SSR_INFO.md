# Информация об использовании SSR в приложении

## Ответ: Частично используется SSR

### Что использует SSR:

1. **Layout (app/layout.tsx):**
   - Серверный компонент (без `'use client'`)
   - SSR используется для метаданных (title, description)
   - HTML оболочка рендерится на сервере

2. **API Routes (app/api/):**
   - Серверный код (не SSR, но работает на сервере)
   - Все API endpoints выполняются на сервере

### Что НЕ использует SSR (Client-Side):

1. **Основная страница (app/page.tsx):**
   - Клиентский компонент (`'use client'`)
   - Данные загружаются на клиенте через `useEffect`
   - Использует `useState`, `useRef` (клиентские хуки)

2. **Страница админки (app/admin/page.tsx):**
   - Клиентский компонент (`'use client'`)
   - Вся логика выполняется на клиенте

### Технические детали:

- **Next.js 16** с App Router
- **Режим сборки:** По умолчанию Next.js использует гибридный подход:
  - Серверные компоненты (layout) - SSR
  - Клиентские компоненты (pages) - Client-Side Rendering с гидратацией
  - API routes - серверный код

### Для Timeweb App Platform:

Приложение работает в **standalone режиме** Next.js (по умолчанию), что означает:
- SSR поддерживается для серверных компонентов
- API routes работают на сервере
- Клиентские компоненты гидратируются на клиенте

**Вывод:** Приложение использует гибридный подход - SSR для layout и метаданных, Client-Side Rendering для основных страниц, и серверные API routes.
