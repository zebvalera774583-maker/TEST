# Timeweb: База данных и хранилище вместо Supabase

## Что может предложить Timeweb:

### 1. Timeweb Cloud Databases (Управляемая PostgreSQL)

**Возможности:**
- ✅ Управляемая PostgreSQL база данных
- ✅ Автоматические бэкапы
- ✅ Масштабирование
- ✅ Российские серверы
- ✅ Работает из России

**Стоимость:** 
- От ~300-500 ₽/мес (нужно уточнить на сайте)

**Где найти:**
- В панели Timeweb Cloud → раздел "Базы данных" или "Databases"

---

### 2. Timeweb Object Storage (Хранилище файлов)

**Возможности:**
- ✅ S3-совместимое хранилище
- ✅ Для фото и файлов
- ✅ Публичные URL
- ✅ Российские серверы

**Стоимость:**
- От ~100-200 ₽/мес за 10-20 ГБ (нужно уточнить)

**Где найти:**
- В панели Timeweb Cloud → раздел "Object Storage" или "Хранилище"

---

## Что нужно сделать для миграции:

### Шаг 1: Создать базу данных в Timeweb

1. Войдите в панель Timeweb Cloud
2. Перейдите в раздел **"Базы данных"** или **"Cloud Databases"**
3. Создайте новую PostgreSQL базу данных
4. Запишите:
   - Host (хост)
   - Port (порт, обычно 5432)
   - Database (имя базы)
   - User (пользователь)
   - Password (пароль)

### Шаг 2: Создать Object Storage bucket

1. В панели Timeweb Cloud перейдите в **"Object Storage"** или **"Хранилище"**
2. Создайте новый bucket (например, "photos" или "test")
3. Настройте публичный доступ (если нужно)
4. Запишите:
   - Endpoint URL
   - Access Key
   - Secret Key
   - Bucket name

### Шаг 3: Миграция данных

1. Экспортировать данные из Supabase
2. Импортировать в Timeweb PostgreSQL
3. Загрузить файлы в Timeweb Object Storage

### Шаг 4: Переписать код

Нужно будет переписать:
- `lib/supabase.ts` → `lib/database.ts` (использовать `pg` или `postgres` вместо Supabase клиента)
- Все API routes для работы с БД
- Загрузку файлов в Object Storage (использовать AWS SDK или S3-совместимый клиент)
