# Решение проблемы с кастомным доменом

## Проблема
- Дефолтный домен Vercel (`test-three-orcin-66.vercel.app`) работает без VPN ✅
- Кастомный домен (`ashot-zebelyan.ru`) не работает без VPN ❌

## Возможные причины

### 1. Проблема с DNS провайдером (nic.ru)
DNS записи могут неправильно распространяться или блокироваться.

**Решение:**
- Проверьте DNS записи в панели nic.ru
- Убедитесь, что A и CNAME записи указаны правильно
- Попробуйте использовать другие DNS серверы (Cloudflare, Google)

### 2. Проблема с SSL сертификатом
Vercel может не выдать SSL сертификат для кастомного домена.

**Решение:**
- В Vercel → Settings → Domains проверьте статус SSL
- Должно быть "Valid" (зеленый)
- Если "Pending" - подождите или перевыпустите сертификат

### 3. Проблема с CDN Vercel для кастомных доменов
Vercel CDN может иметь проблемы с кастомными доменами в России.

**Временное решение:**
- Использовать VPN
- Или использовать дефолтный домен Vercel

**Долгосрочное решение:**
- Перенести на российский хостинг с кастомным доменом

## Что проверить в Vercel

1. **Settings → Domains:**
   - Статус домена `ashot-zebelyan.ru` должен быть "Valid" (зеленый)
   - Статус SSL должен быть "Valid"
   - Если "Pending" - подождите или перевыпустите

2. **DNS записи в nic.ru:**
   - A запись для `ashot-zebelyan.ru` → IP от Vercel (76.76.21.21 или другой)
   - CNAME запись для `www.ashot-zebelyan.ru` → `cname.vercel-dns.com`

## Альтернативное решение

Если проблема с DNS/SSL не решается, можно:
1. Использовать дефолтный домен Vercel (работает без VPN)
2. Настроить редирект с кастомного домена на дефолтный
3. Или перенести на российский хостинг
