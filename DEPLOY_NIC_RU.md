# Деплой на хостинг nic.ru

## Важно: Требования для Next.js

Для Next.js приложения нужен хостинг с поддержкой:
- **Node.js** (версия 18+ или 20+)
- **npm/node** для сборки и запуска
- **PM2** или другой процесс-менеджер (рекомендуется)
- **Nginx** для проксирования (рекомендуется)

## Проверка хостинга nic.ru

### 1. Тип хостинга

nic.ru обычно предлагает:
- **Shared хостинг** (PHP/статический) - НЕ подходит для Next.js ❌
- **VPS/VDS** - подходит для Next.js ✅
- **Managed хостинг с Node.js** - подходит для Next.js ✅

### 2. Что нужно проверить:

1. **Есть ли доступ к SSH?**
   - Next.js нужно запускать через командную строку
   - Нужен SSH доступ к серверу

2. **Есть ли Node.js?**
   ```bash
   node --version
   npm --version
   ```
   - Нужна версия Node.js 18+ или 20+

3. **Есть ли возможность устанавливать пакеты?**
   - Нужно установить PM2: `npm install -g pm2`
   - Нужно установить зависимости: `npm install`

4. **Есть ли Nginx или другой веб-сервер?**
   - Для проксирования запросов к Next.js (порт 3000)

### 3. Если это VPS от nic.ru

Если nic.ru предоставляет VPS, процесс будет такой же, как для Yandex Cloud:
- Установить Node.js
- Установить Nginx
- Установить PM2
- Склонировать проект
- Собрать и запустить
- Настроить Nginx
- Настроить домен

### 4. Если это Shared хостинг

Shared хостинг от nic.ru НЕ подойдет, так как:
- Нет доступа к Node.js
- Нет SSH доступа
- Нет возможности запускать процессы

## Рекомендации

1. **Уточните тип хостинга у nic.ru:**
   - Это VPS/VDS?
   - Или shared хостинг?
   - Есть ли поддержка Node.js?

2. **Если это VPS:**
   - Можно использовать инструкции из `DEPLOY_YANDEX_CLOUD.md`
   - Адаптируем под сервер nic.ru

3. **Если это shared хостинг:**
   - Рассмотрите альтернативы: Yandex Cloud, Timeweb, Selectel
   - Или статический экспорт Next.js (если проект позволяет)
