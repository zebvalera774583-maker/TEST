# Настройка DNS для App Platform в Timeweb

## Текущая ситуация:

В DNS записях домена `ashot-zebelyan.ru` видна:
- **A запись:** `ashot-zebelyan.ru` → `147.45.99.106`

**Проблема:** Эта A запись указывает на старый IP адрес, а не на App Platform приложение.

---

## Решение:

### Вариант 1: Привязать домен через настройки приложения (РЕКОМЕНДУЕТСЯ)

**Это самый простой способ:**

1. **Перейдите в настройки приложения:**
   - **App Platform** → **ashot-zebelyan** → **"Настройки"** → **"Домены"**

2. **Добавьте домен там:**
   - Нажмите **"Редактировать"**
   - Добавьте `ashot-zebelyan.ru`
   - Добавьте `www.ashot-zebelyan.ru`

3. **Timeweb автоматически:**
   - Покажет правильные DNS записи (если нужно)
   - Или настроит DNS автоматически (если домен на NS-серверах Timeweb)
   - Выпустит SSL сертификат

### Вариант 2: Изменить DNS записи вручную

Если нужно изменить DNS записи вручную:

1. **Удалите старую A запись:**
   - В DNS записях найдите A запись с IP `147.45.99.106`
   - Удалите её (или измените значение)

2. **Добавьте CNAME запись:**
   - Нажмите **"Добавить запись"**
   - **Тип:** CNAME
   - **Хост:** `ashot-zebelyan.ru` (или `@`)
   - **Значение:** `zebvalera774583-maker-test-16b8.twc1.net` (временный домен вашего приложения)
   - **TTL:** 600 (или по умолчанию)

3. **Для www:**
   - **Тип:** CNAME
   - **Хост:** `www`
   - **Значение:** `zebvalera774583-maker-test-16b8.twc1.net`
   - **TTL:** 600

**Но лучше использовать Вариант 1** - Timeweb сам настроит правильные записи!

---

## Важно:

### Если домен на NS-серверах Timeweb:

Если домен размещен на NS-серверах Timeweb (как видно на скриншоте):
- DNS записи управляются автоматически
- Нужно только привязать домен к приложению через настройки приложения
- Timeweb сам настроит правильные записи

### Если домен на NS-серверах nic.ru:

Если домен на NS-серверах nic.ru:
- Нужно добавить DNS записи в nic.ru
- Timeweb покажет, какие именно записи нужны
- Обычно это CNAME записи на временный домен приложения
