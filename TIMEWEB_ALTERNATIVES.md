# Альтернативы Dockerfile на Timeweb (экономия)

## Проблема:
Тариф с Dockerfile стоит **800 ₽/мес**, что дорого.

## Варианты решения:

### ✅ Вариант 1: Оставить текущий деплой (БЕСПЛАТНО/дешевле)

**Вопрос:** А работает ли ваше текущее приложение на Timeweb БЕЗ Dockerfile?

Если текущее приложение работает (даже если поддержка рекомендовала Dockerfile), можно оставить как есть. Возможно, Dockerfile рекомендован для оптимизации, но не обязателен.

**Что проверить:**
1. Откройте ваше текущее приложение на Timeweb
2. Проверьте, работают ли API routes (`/api/admin/auth`, `/api/admin/photos`)
3. Проверьте загрузку фото через админку
4. Если всё работает — зачем платить больше?

### ✅ Вариант 2: Использовать Timeweb VPS (200-300 ₽/мес)

**Timeweb VPS** (виртуальный сервер):
- Стоимость: от **200-300 ₽/мес** (минимальный тариф)
- Нужна ручная настройка (Node.js, PM2, Nginx)
- Полный контроль
- Инструкция уже есть: `DEPLOY_TIMEWEB_VPS.md`
- Без авто-деплоя (но можно настроить через GitHub Actions)

### ✅ Вариант 3: Уточнить у поддержки Timeweb

Спросить у поддержки:
- Можно ли использовать Next.js с API routes БЕЗ Dockerfile на обычном тарифе?
- Есть ли более дешевый тариф для таких приложений?
- Почему нужен именно Dockerfile? В чем реальная проблема?

### ❌ Вариант 4: Другие платформы

**Yandex Cloud:**
- Дороже (400-600 ₽/мес+)
- Инструкция: `DEPLOY_YANDEX_CLOUD.md`

**Vercel:**
- Бесплатно, но не работает без VPN (как вы и обнаружили)

## Рекомендация:

1. **Сначала проверьте** — работает ли ваше текущее приложение на Timeweb БЕЗ Dockerfile?
   - Если работает — оставьте как есть, не платите больше!
   
2. Если не работает — рассмотрите **Timeweb VPS** (200-300 ₽/мес) с ручной настройкой
   - Экономия 500-600 ₽/мес
   - Нужна будет ручная настройка (но инструкция есть)

3. Если нужен авто-деплой и текущий деплой не работает — тогда только Dockerfile (800 ₽/мес)
