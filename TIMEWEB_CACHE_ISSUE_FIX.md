# Проблема: Кэш не очищается, в инкогнито работает

## Ситуация:
- ✅ Временный домен работает правильно
- ✅ В режиме инкогнито кастомный домен работает правильно
- ❌ После очистки кэша все равно открывается админка

---

## Возможные причины:

### 1. Cookies или localStorage для домена

**Проблема:**
- В cookies или localStorage может быть сохранена информация о редиректе
- Очистка кэша не удаляет cookies и localStorage

**Решение:**
1. Откройте `https://ashot-zebelyan.ru`
2. Нажмите `F12` (Developer Tools)
3. Перейдите на вкладку **"Application"** (или "Хранилище")
4. В левом меню найдите:
   - **"Cookies"** → `https://ashot-zebelyan.ru`
   - **"Local Storage"** → `https://ashot-zebelyan.ru`
5. Удалите все cookies и данные Local Storage для этого домена
6. Обновите страницу

### 2. Проблема с настройками домена в Timeweb

**Возможная причина:**
- В настройках домена может быть настроен редирект или прокси
- Или домен неправильно привязан к приложению

**Проверка:**
- App Platform → ashot-zebelyan → "Настройки" → "Домены"
- Проверьте настройки домена `ashot-zebelyan.ru`
- Нет ли там настроек редиректа или прокси?

### 3. Проблема с DNS/прокси на уровне Timeweb

**Возможная причина:**
- Timeweb может кэшировать редиректы на уровне прокси
- Или DNS записи указывают не туда

**Проверка:**
- Проверьте DNS записи в nic.ru
- Должны указывать на `147.45.99.106`

---

## Решение:

### Шаг 1: Удалить Cookies и Local Storage

1. Откройте `https://ashot-zebelyan.ru`
2. Нажмите `F12` (Developer Tools)
3. Перейдите на вкладку **"Application"** (или "Хранилище" / "Storage")
4. В левом меню:
   - **"Cookies"** → `https://ashot-zebelyan.ru` → Удалите все
   - **"Local Storage"** → `https://ashot-zebelyan.ru` → Удалите все
   - **"Session Storage"** → `https://ashot-zebelyan.ru` → Удалите все
5. Закройте все вкладки с этим доменом
6. Откройте `https://ashot-zebelyan.ru` заново

### Шаг 2: Проверить вкладку Network

1. Откройте `https://ashot-zebelyan.ru`
2. Нажмите `F12` → вкладка **"Network"** (Сеть)
3. Обновите страницу (F5)
4. Посмотрите на первый запрос к `ashot-zebelyan.ru`:
   - Какой статус? (200, 301, 302, 404?)
   - Есть ли редирект? (301, 302)
   - Куда перенаправляет? (Location header)
   - Какой URL в итоге загружается?

### Шаг 3: Проверить настройки домена в Timeweb

1. App Platform → ashot-zebelyan → "Настройки" → "Домены"
2. Нажмите на домен `ashot-zebelyan.ru` (или "Редактировать")
3. Проверьте:
   - Правильно ли привязан к приложению?
   - Нет ли настроек редиректа?
   - Нет ли настроек прокси?
   - Какой статус SSL?

### Шаг 4: Проверить DNS

В командной строке выполните:
```bash
nslookup ashot-zebelyan.ru
```

Должен показать: `147.45.99.106`
