# Проблема: Главная страница показывает админку

## Ситуация:
При заходе на `ashot-zebelyan.ru` открывается страница админки вместо главной страницы с галереей.

---

## Возможные причины:

### 1. Ошибка при загрузке данных из Supabase

**Проблема:**
- Главная страница пытается загрузить фото из Supabase
- Если возникает ошибка, возможно происходит редирект или показывается админка

**Проверка:**
- Откройте консоль браузера (F12) → вкладка "Console"
- Посмотрите, есть ли ошибки подключения к Supabase

### 2. Проблема с переменными окружения

**Проблема:**
- Переменные окружения не настроены правильно
- Supabase клиент не может подключиться
- Происходит ошибка

**Проверка:**
- App Platform → ashot-zebelyan → "Настройки" → "Переменные окружения"
- Убедитесь, что все переменные добавлены

### 3. Проблема с роутингом Next.js

**Проблема:**
- Возможно, Next.js неправильно обрабатывает маршрут `/`
- Или есть проблема с файлом `app/page.tsx`

**Проверка:**
- Проверьте логи приложения в Timeweb
- Посмотрите, нет ли ошибок при загрузке страницы

### 4. Кэш браузера

**Проблема:**
- Браузер кэшировал старую версию
- Показывает админку из кэша

**Решение:**
- Очистите кэш браузера
- Или откройте в режиме инкогнито

---

## Диагностика:

### Шаг 1: Проверить в режиме инкогнито

1. Откройте браузер в режиме инкогнито (Ctrl+Shift+N)
2. Откройте `https://ashot-zebelyan.ru`
3. Что открывается? Главная страница или админка?

### Шаг 2: Проверить консоль браузера

1. Откройте `https://ashot-zebelyan.ru`
2. Нажмите `F12` (открыть Developer Tools)
3. Перейдите на вкладку **"Console"**
4. Посмотрите, есть ли ошибки (красные сообщения):
   - Ошибки подключения к Supabase?
   - Ошибки загрузки данных?
   - Другие ошибки?

### Шаг 3: Проверить логи приложения

1. App Platform → ashot-zebelyan → **"Логи приложения"**
2. Посмотрите, есть ли ошибки:
   - Ошибки подключения к Supabase?
   - Ошибки загрузки данных?
   - Ошибки работы приложения?

### Шаг 4: Проверить переменные окружения

1. App Platform → ashot-zebelyan → **"Настройки"** → **"Переменные окружения"**
2. Проверьте, что все переменные добавлены:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `ADMIN_PASSWORD`
   - `NODE_ENV=production`

### Шаг 5: Проверить временный домен

1. Откройте временный домен: `https://zebvalera774583-maker-test-16b8.twc1.net`
2. Что открывается? Главная страница или админка?
3. Если временный домен работает правильно - проблема может быть в DNS или кэше

## Возможные решения:

### Если видите страницу входа в админку:

**Это странно** - главная страница не должна показывать админку.

**Возможные причины:**
- Ошибка при загрузке главной страницы
- Next.js показывает страницу ошибки
- Проблема с роутингом

**Решение:**
- Проверьте консоль браузера (F12) - там должны быть ошибки
- Проверьте логи приложения в Timeweb

### Если видите пустую страницу:

**Возможная причина:**
- Ошибка при загрузке данных
- JavaScript не выполняется

**Решение:**
- Проверьте консоль браузера (F12)
- Проверьте переменные окружения

### Если видите ошибку:

**Решение:**
- Прочитайте текст ошибки
- Проверьте, что она означает
- Исправьте проблему
