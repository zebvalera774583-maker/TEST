# Исправление ошибки SSL: ERR_SSL_PROTOCOL_ERROR

## Проблема:
При открытии `ashot-zebelyan.ru` возникает ошибка: **ERR_SSL_PROTOCOL_ERROR**

**Это означает:**
- Домен не настроен правильно в Timeweb
- Или SSL сертификат еще не выпущен/не настроен
- Или DNS записи не обновились

---

## Решение:

### Шаг 1: Проверить настройки домена в Timeweb

1. **Войдите в панель Timeweb Cloud**
2. **Перейдите в ваше приложение** `ashot-zebelyan`
3. **Перейдите в раздел "Настройки"** → **"Домены"** или **"Domains"**
4. **Проверьте:**
   - Добавлен ли домен `ashot-zebelyan.ru`?
   - Добавлен ли домен `www.ashot-zebelyan.ru`?
   - Какой статус у доменов? (Должен быть "Активен" или "Valid")

### Шаг 2: Проверить DNS записи в nic.ru

1. **Войдите в панель управления доменом в nic.ru**
2. **Перейдите в "Управление DNS-записями"** (или "DNS-master")
3. **Проверьте записи:**
   - Должны быть CNAME записи, которые показал Timeweb
   - Или A записи с IP адресом Timeweb

### Шаг 3: Дождаться выпуска SSL сертификата

**Важно:** После добавления домена в Timeweb:
- SSL сертификат выпускается автоматически
- Это может занять **10-30 минут** (иногда до часа)
- Нужно дождаться завершения процесса

**Как проверить:**
- В настройках домена в Timeweb должен быть статус SSL сертификата
- Обычно показывается: "Выпускается..." или "Активен"

### Шаг 4: Использовать временный URL для проверки

**Пока SSL сертификат выпускается, можно проверить работу приложения:**

1. **В панели Timeweb найдите временный URL:**
   - Обычно это: `ashot-zebelyan-xxxxx.timeweb.cloud`
   - Или в настройках приложения должен быть URL

2. **Откройте временный URL в браузере:**
   - Он должен работать сразу (SSL уже настроен для временного домена)
   - Проверьте работу сайта и админки

---

## Возможные причины ошибки:

### 1. Домен не добавлен в Timeweb

**Решение:**
- Добавьте домен в настройках приложения
- Дождитесь выпуска SSL сертификата

### 2. DNS записи не обновились

**Решение:**
- Проверьте DNS записи в nic.ru
- Подождите 10-30 минут для распространения DNS
- Очистите кэш DNS на компьютере: `ipconfig /flushdns`

### 3. SSL сертификат еще не выпущен

**Решение:**
- Подождите 10-30 минут (иногда до часа)
- Проверьте статус SSL в настройках домена в Timeweb

### 4. Неправильные DNS записи

**Решение:**
- Убедитесь, что DNS записи указывают на Timeweb
- Используйте те записи, которые показал Timeweb при добавлении домена

---

## Пошаговая инструкция:

### 1. Добавить домен в Timeweb (если еще не добавлен):

1. В панели Timeweb → ваше приложение → **"Настройки"** → **"Домены"**
2. Нажмите **"Добавить домен"**
3. Введите: `ashot-zebelyan.ru`
4. Нажмите **"Добавить"**
5. Повторите для `www.ashot-zebelyan.ru`
6. Timeweb покажет DNS записи, которые нужно добавить

### 2. Добавить DNS записи в nic.ru:

1. Войдите в панель nic.ru
2. Перейдите в **"Управление DNS-записями"**
3. Удалите старые записи (если были для Vercel)
4. Добавьте новые записи, которые показал Timeweb:
   - Обычно это CNAME записи
   - Или A записи с IP адресом

### 3. Дождаться:

- **10-30 минут** для распространения DNS
- **10-30 минут** для выпуска SSL сертификата

### 4. Проверить:

- Откройте `https://ashot-zebelyan.ru`
- Если ошибка сохраняется - проверьте статус SSL в Timeweb

---

## Альтернатива: Проверить через временный URL

**Пока домен настраивается:**

1. Найдите временный URL в панели Timeweb
2. Откройте его в браузере
3. Проверьте работу сайта
4. После настройки домена всё будет работать на `ashot-zebelyan.ru`

---

## Если ошибка сохраняется:

1. **Проверьте логи приложения** в Timeweb
2. **Проверьте статус SSL** в настройках домена
3. **Обратитесь в поддержку Timeweb** - они помогут с настройкой SSL

---

**Главное:** Дождитесь выпуска SSL сертификата (10-30 минут) и проверьте настройки DNS!
