# Проблема: Главная страница перенаправляет на админку

## Проблема:
При заходе на `ashot-zebelyan.ru` открывается страница админки вместо главной страницы с галереей.

---

## Возможные причины:

### 1. Проблема с кэшем браузера

**Решение:**
- Очистите кэш браузера
- Или откройте сайт в режиме инкогнито
- Или используйте другой браузер

### 2. Проблема с роутингом Next.js

**Возможная причина:**
- Ошибка при загрузке главной страницы
- Next.js перенаправляет на `/admin` при ошибке

**Решение:**
- Проверьте логи приложения в Timeweb
- Проверьте, нет ли ошибок при загрузке данных

### 3. Проблема с переменными окружения

**Возможная причина:**
- Переменные окружения не настроены правильно
- Приложение не может загрузить данные из Supabase
- Происходит ошибка и редирект

**Решение:**
- Проверьте переменные окружения в Timeweb
- Убедитесь, что все переменные добавлены

### 4. Ошибка при загрузке фото

**Возможная причина:**
- Ошибка при загрузке фото из Supabase
- Приложение показывает ошибку или редиректит

**Решение:**
- Проверьте логи приложения
- Проверьте подключение к Supabase

---

## Диагностика:

### Шаг 1: Проверить в режиме инкогнито

1. Откройте браузер в режиме инкогнито
2. Откройте `https://ashot-zebelyan.ru`
3. Что открывается? Главная страница или админка?

### Шаг 2: Проверить логи приложения

1. В панели Timeweb: App Platform → ashot-zebelyan → **"Логи приложения"**
2. Посмотрите, есть ли ошибки:
   - Ошибки подключения к Supabase?
   - Ошибки загрузки данных?
   - Другие ошибки?

### Шаг 3: Проверить переменные окружения

1. App Platform → ashot-zebelyan → **"Настройки"** → **"Переменные окружения"**
2. Проверьте, что все переменные добавлены:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `ADMIN_PASSWORD`
   - `NODE_ENV=production`

### Шаг 4: Проверить консоль браузера

1. Откройте сайт `https://ashot-zebelyan.ru`
2. Нажмите `F12` (открыть Developer Tools)
3. Перейдите на вкладку **"Console"**
4. Посмотрите, есть ли ошибки (красные сообщения)

---

## Возможные решения:

### Решение 1: Очистить кэш браузера

1. Нажмите `Ctrl+Shift+Delete`
2. Выберите "Кэш" или "Cached images and files"
3. Нажмите "Очистить"
4. Обновите страницу

### Решение 2: Проверить URL

**Убедитесь, что вы открываете правильный URL:**
- ✅ Правильно: `https://ashot-zebelyan.ru`
- ✅ Правильно: `https://ashot-zebelyan.ru/`
- ❌ Неправильно: `https://ashot-zebelyan.ru/admin`

### Решение 3: Проверить работу Supabase

1. Проверьте, работает ли Supabase
2. Откройте панель Supabase
3. Проверьте, что таблицы созданы
4. Проверьте, что данные есть
